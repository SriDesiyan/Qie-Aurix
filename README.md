## The Problem

Web3 gives users freedom, but not enough protection.

Today, users manage assets across multiple chains, wallets, protocols, and liquidity systems without a unified safety layer. That creates real risk:

- assets become fragmented across networks
- smart contracts can expose users to vulnerabilities
- mistaken token transfers can be irreversible
- private key management remains a single point of failure
- recovery options are limited
- family inheritance planning is difficult and insecure
- protection tools are usually reactive instead of proactive

Most tools in Web3 focus on speculation, yield, and trading.

Very few focus on resilience.

## Core Features

### Trust Profile Engine
Builds a dynamic trust profile from verified identity signals and on-chain behavior.  
This is the foundation for personalized protection.

### Resilience Score
Generates a transparent score from **0 to 1000** based on asset stability, recovery readiness, family safety, and contract integrity.

### Guardian Mode™
One-click protection mode that can activate reserve allocation, safety policies, family vaults, and monitoring flows.

### Recovery Layer
Supports verified recovery workflows for mistakenly transferred tokens from system vaults.

### Family Vaults
Creates secure inheritance and multi-generational vault structures with controlled access and release logic.

### Safety Audit Anchor
Anchors audit summaries and system integrity records on-chain for traceability and transparency.

### Protection Policies
Lets users define custom rules for automated financial protection, such as rebalancing or reserve allocation.

# Oracle API launches at http://localhost:8000

cd apps/aurix-contracts
npm install
npx hardhat compile
Security Principles

Aurix is built around a few non-negotiable safety rules:

Cryptographic Signatures: All recovery requests require claimant wallet signatures verified by on-chain ECDSA checks.
Anti-Replay Protections: Nonces are incorporated into policies to prevent transaction replication.
Role Controls: Core admin actions and pause triggers are guarded by strict access controls.
Immutable Anchoring: Audits are anchored via safety logs to prevent retro-active modification of scores.
Timelock Safe-guards: Heir distributions are locked behind configurable release delay parameters.

Security is not an add-on. It is the core product.
